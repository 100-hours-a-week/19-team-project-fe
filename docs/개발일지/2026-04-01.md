# 2026-04-01 개발일지

## 주제

온보딩 프로필 폼 React Hook Form(RHF) 도입 준비 및 전략 문서화

---

## 1) 배경

- 온보딩 프로필 폼은 현재 `useState` 중심으로 상태/검증/제출 로직이 분산되어 있다.
- 필드/검증이 증가할수록 변경 지점이 많아지고, 제출 가능 조건과 실제 검증 로직이 이중화되고 있다.
- 향후 설문조사/확장 폼까지 고려하면 폼 상태 관리의 표준화가 필요하다.

---

## 2) 오늘 목표

- RHF 도입의 정의/핵심 개념을 팀 문서로 정리한다.
- 현재 온보딩(설문조사 확장 전) 구현의 문제 지점을 코드 기준으로 기록한다.
- 온보딩 프로필 폼 우선 전환을 위한 실행 계획을 단계별로 정리한다.

---

## 3) 오늘 작업 내용

### 3-1. RHF 전략 문서 신규 작성

파일:

- `docs/strategy/rhf.md` (신규)

정리한 내용:

- RHF 정의
- 핵심 개념(`useForm`, `register`, `Controller`, `formState`, `handleSubmit`)
- Resolver/스키마 검증 적용 관점
- 현재 온보딩 구현 발췌 코드 블록
- 온보딩 우선 전환 범위/원칙/단계별 계획

---

### 3-2. 온보딩 현재 구조(설문조사 이전) 기준선 정리

문서에 포함한 핵심 포인트:

- 상태 분산: `selectedJob`, `selectedCareer`, `selectedTech`, `nickname`, 약관 동의 상태가 별도 `useState`
- 수동 검증: 제출 훅에서 조건별 `setSubmitError` 분기
- 중복 조건: `isSubmitDisabled`와 submit 검증 로직의 중첩

목적:

- 도입 이전 기준선을 명확히 하여, 전환 후 개선 효과(복잡도/유지보수성)를 비교 가능하게 만듦
