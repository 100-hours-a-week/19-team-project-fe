# 2026-02-23 개발일지

## 주제

Firebase Cloud Messaging(FCM) 기반 웹 푸시 알림 연동 및 알림 조회 UI 구축

---

## 1) 목표

- 로그인 사용자 기준으로 웹 푸시 토큰을 발급/등록한다.
- 백그라운드 푸시 수신 시 OS 알림이 표시되도록 서비스워커를 구성한다.
- 알림 목록 조회/읽음 처리를 프론트에서 사용할 수 있도록 BFF + Entity API를 만든다.
- 홈 헤더 알림 버튼에서 알림 전용 페이지로 이동하고, 서버 상태 기반으로 배지/목록을 갱신한다.

---

## 2) 환경/설정 작업

### 2-1. 환경 변수 정리

FCM 웹 설정값을 `.env.local`에 반영:

- `NEXT_PUBLIC_FIREBASE_API_KEY`
- `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN`
- `NEXT_PUBLIC_FIREBASE_PROJECT_ID`
- `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID`
- `NEXT_PUBLIC_FIREBASE_APP_ID`
- `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET`
- `NEXT_PUBLIC_FIREBASE_VAPID_KEY`

주의:

- `service_account` JSON(private key 포함)은 프론트에 사용하지 않음.
- 프론트는 Web App 공개 설정값 + VAPID public key만 사용.

### 2-2. 의존성

- `firebase` 패키지 추가 (`package.json`, `pnpm-lock.yaml`)

---

## 3) 아키텍처 결정

### 3-1. 역할 분리

- Service Worker: 백그라운드 수신/OS 알림 표시/클릭 포워딩
- TanStack Query: 알림 목록/읽음 상태/unreadCount 서버 상태 단일 소스
- UI 컴포넌트: 페이지 렌더링 및 사용자 액션

### 3-2. API 호출 경로

기존 프로젝트 패턴을 유지하기 위해 클라이언트에서 백엔드 직접 호출 대신 BFF 경유:

- `client -> /bff/notifications/* -> upstream /api/v2/notifications*`
